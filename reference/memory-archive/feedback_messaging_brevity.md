Slack drafts are consistently too long and too formal. The user rewrites them down to ~50% length.

**Why:** The user's actual style is 2-3 sentences per message, leading with a specific proposal in tentative framing ("My first thought is to change X from Y to Z"). Claude's drafts tend toward 4 paragraphs with context-setting preamble and industry jargon the audience doesn't need.

**How to apply:**
- Cut first: write the draft, then halve it. Remove any sentence that explains what the audience already knows.
- Lead with the concrete proposal, not background.
- Use tentative framing ("my first thought", "I'd suggest") not prescriptive ("a more X approach would be").
- Name exact values ("from Write to No permission") not abstract concepts ("set to no access").
- Drop industry-speak ("least-privilege best practices") — just state what happens ("stays private unless you grant access").
- Suggest splitting into multiple short messages when covering several points.
