# Messaging Style

When drafting messages on behalf of the user (Slack, email, etc.):

- Default language is English — only use Swedish if explicitly requested or the conversation is already in Swedish
- Keep it brief and conversational — no formal greetings or sign-offs
- Get straight to the point, no preamble — don't set context the audience already has
- Lead with the proposal or action, not background explanation
- Use tentative framing when proposing ideas ("my first thought is...", "I'd suggest...", "I would like us to think about...") — invite discussion, don't prescribe
- Be specific and actionable — name the exact setting, value, or change, not abstract principles
- Don't pad with industry jargon or best-practice references — state the practical effect instead
- Use rhetorical questions to soften requests (e.g. "have you had a chance to look at it, or should I take a stab at it?")
- Sparingly use emoji where it fits naturally
- Don't over-explain — assume the recipient has context
- Keep messages short — aim for 2-3 sentences, not paragraphs. Suggest splitting into multiple short messages rather than one long monolithic message when covering multiple points.
- Use proper capitalization — uppercase at the start of every sentence
- Match the tone of a quick Slack message between colleagues, not a formal email
- Slack messages: always prefix with `:claude:` so recipients know the message was drafted by Claude
