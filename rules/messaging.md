# Messaging Style

When drafting messages on behalf of the user (Slack, email, etc.):

- Default language is English — only use Swedish if explicitly requested or the conversation is already in Swedish
- Keep it brief and conversational — no formal greetings or sign-offs
- Get straight to the point, no preamble — don't set context the audience already has
- Lead with the proposal or action, not background explanation
- Use tentative framing when proposing ideas ("my first thought is...", "I'd suggest...", "I would like us to think about...") — invite discussion, don't prescribe
- Soften the opener — single-word openers like "Sure", "Yeah", "Happy to" plus a hedged verb ("I think", "probably") land warmer than "Yes — we can…", which reads transactional
- Keep conversational asides ("as always", "probably", "should be simple enough") — these signal colleague-to-colleague tone; don't strip them in the name of concision
- Name the hard part directly in prose instead of enumerating concerns as a structured list ("The hard part, as always, is probably X" > "A couple of things we want to check: (1)… (2)…")
- "I" vs "we" is situational, not blanket — use "I" for personal action/ownership ("I will ping the team", "I'm off"), "we" for team capability or shared action ("we can help", "we would configure as code"). Mix them the way a person would
- Be specific and actionable — name the exact setting, value, or change, not abstract principles
- Don't pad with industry jargon or best-practice references — state the practical effect instead
- Cut parenthetical qualifiers that restate the recipient's own ask — they wrote the requirements; don't re-specify them in parens
- Use rhetorical questions to soften proposals — turn statements into questions ("maybe there's something worth striking while it's hot?" beats "there's a window worth striking", "maybe the list can be managed as a Google Group?" beats a long declarative explanation)
- Personal context warms up timing — "Personally I'm off next week" explains deferral humanely and beats a bare "we'll come back later"
- Emoji are part of the voice, not decoration — a playful opener (`:big-frog:`) or softener (`:slightly_smiling_face:`, `:sunny:`) fits naturally in opinion/pitch messages. Use more liberally than "sparingly" when the tone is personal rather than transactional. Don't mechanically mirror the recipient's emoji — your emoji reflects your voice, not theirs
- Italic `_word_` for tentative emphasis on scope/time qualifiers is optional — plain "next week" often reads better than "_next week_" in light replies; reserve italic for when the emphasis is genuinely load-bearing (`_soon_`, `_today!_`)
- Don't over-explain — assume the recipient has context
- Keep messages short — aim for 2-3 sentences, not paragraphs. Suggest splitting into multiple short messages rather than one long monolithic message when covering multiple points. Exception: opinion/pitch messages can run longer when there's real substance
- Use proper capitalization — uppercase at the start of every sentence
- Match the tone of a quick Slack message between colleagues, not a formal email
- Slack messages: always prefix with `:claude:` so recipients know the message was drafted by Claude

## Translating or broadening a user's existing draft

- Preserve social openers (apologies, acknowledgments of tension) and personal stakes ("I was on both sides of that") — if the user wrote them, they matter. Don't drop them thinking the new audience doesn't need them
- When broadening from 1:1 to team, strip 1:1-specific political context (named people being difficult, back-channel conversations) but keep the emotional framing intact
- Don't name-drop specific teams/departments in proposals when an inclusive "us"/"we" works — "enable non-engineers" beats "PlatEng should enable non-engineers"
- Broaden the evidence base — if the original leans on one person's asks, look for a framing that acknowledges wider signal ("X is not the only one who's asked")
- Include epistemic humility in opinion messages ("I don't know", "nothing here is clear", "just a feeling this is important") — don't over-polish opinion into confident assertion. The user's voice has more uncertainty than clean drafts tend to convey
