# Tooling

- Python: always use `uv run` to run scripts (e.g. `uv run main.py`)
- `uv sync`/`uv run` rewrites `uv.lock` source registries to the locally-configured index (e.g. `pypi.org/simple` ‚Üí `pypi.kognic.io/simple`) when the committed lock used a different index. This is local-env noise, not a real dependency change ‚Äî `git checkout uv.lock` (or unstage it) before committing unless you intentionally changed deps. Check `git diff uv.lock` after any `uv` invocation in a repo you'll commit to.
- Sandbox cache redirects: many tools fail in the command sandbox when writing their default cache under `~/.cache` or `~/Library/Caches` (`Operation not permitted` / `failed to initialize cache`). These caches are now redirected globally via the `env` block in `settings.json` (`UV_CACHE_DIR`, `GOCACHE`, `XDG_CACHE_HOME`, `npm_config_cache` ‚Üí fixed `/tmp/claude/*` paths, which are sandbox-writable and stable across sandboxed/non-sandboxed calls). **Do not prepend per-command cache env prefixes** for `uv`, `go`, `gh`, or `npm`/`npx` ‚Äî run them bare (`go test ./...`, `gh run view ‚Ä¶`) so the `permissions.allow` patterns match instead of falling through to the auto-mode classifier. Only add an inline cache env var for a tool not covered by the global block, and prefer a fixed `/tmp/claude/<tool>-cache` path over `$TMPDIR` (see the cross-call note below).
- Provider-plugin tools under the sandbox: `terraform`/`tofu test` (and any command that must launch the provider ‚Äî `plan`, `apply`, `providers schema`) fail with `Unrecognized remote plugin message` / `Failed to read any lines from plugin's stdout` ‚Äî the sandbox blocks the provider binary's go-plugin handshake. This is *not* a cache issue; rerun with the sandbox disabled. `init` works sandboxed (download only), so init in-sandbox, then run the provider-launching command with the sandbox off.
- `$TMPDIR` resolves to a *different directory* in sandboxed vs non-sandboxed Bash calls (`/tmp/claude-501/...` vs `/var/folders/.../T/`). A file written to `$TMPDIR` with the sandbox off is invisible to a later sandboxed call that re-expands `$TMPDIR` (and vice versa). When a temp file must cross between calls with different sandbox settings ‚Äî e.g. dump `tofu providers schema -json` sandbox-off, then `jq` it sandboxed ‚Äî note the absolute path from the first call's output and use it verbatim in the next.
- Before relying on version-specific behaviour of a tool the CI runs (e.g. a `terraform`/`tofu` feature like `replace_triggered_by`), check the version the CI/runner actually pins ‚Äî local and CI versions can differ and silently change behaviour. Reproduce against the CI version (download the exact binary if needed) before shipping a fix that depends on it.
- Regenerate an OpenTofu/Terraform lock file (`.terraform.lock.hcl`) without backend access: `tofu init -backend=false` installs providers + modules without remote-state creds, and `tofu providers lock -platform=‚Ä¶ ‚Ä¶` never touches the backend at all. Use both to regenerate a lock from a laptop that can't reach the remote state backend (e.g. an air-gapped China S3 backend): run `init -backend=false` first (installs modules, so transitive provider requirements like `hashicorp/time` are discovered), then `providers lock` listing every consuming platform ‚Äî `-platform=linux_amd64` for the CI runner **plus** `-platform=darwin_arm64` for laptops (a laptop-only lock omits the runner's `h1:`). Private git module sources need the network sandbox off (registry + GitHub fetch). On macOS the `osxkeychain` git credential helper already authenticates `github.com/annotell/...`, so `init -backend=false` usually fetches the modules with **no token env at all** ‚Äî try it bare first. If auth genuinely fails, add `GITHUB_TOKEN=$(gh auth token)` using the substitution form; never paste a literal resolved `gho_...` token into the command/env ‚Äî the auto-mode classifier blocks that as credential leakage. When you are *changing* a pinned provider version (especially a **downgrade**), plain `init -backend=false` won't move it and `providers lock` errors (`locked provider ‚Ä¶ does not match configured version constraint; must use tofu init -upgrade`). Run `init -backend=false -upgrade` first, then `providers lock`. `-upgrade` only moves providers whose pin actually changed ‚Äî sibling providers that are exact-pinned (`version = "3.7.2"`) stay put ‚Äî so it's safe to run across many states for a single-provider bump/downgrade. Verify afterward that the lock diff touches only the intended provider's `version`/`hashes` (`git diff '*.terraform.lock.hcl' | grep -E '^[+-]\s*version'`).
- Generated files that CI verifies via "regenerate + `git diff --exit-code`" (terraform-docs READMEs, OpenAPI/codegen output, lock files) must be produced by the generator, never hand-written or hand-edited ‚Äî a hand-made version won't match byte-for-byte and fails the check, often via an invisible diff (whole file shows changed with identical-looking content = line endings; e.g. `make docs` runs terraform-docs through docker `-t`, so its READMEs are CRLF). Run the project's generator (`make docs`, the cached docker image, etc.); if you can't, say so and leave it for the generator rather than guessing. Confirm with `file <path>` / a line-ending check when a diff looks like a phantom.
- Helm: read chart values with `helm show values`
- Use built-in tools for all file operations ‚Äî never fall back to shell equivalents:
  - **Read** not `cat`, `head`, `tail`, `sed -n` ‚Äî use `offset`/`limit` params to read specific line ranges
  - **Glob** not `find`, `ls`
  - **Grep** not `grep`, `rg`, `awk`
  - **Edit** not `sed`, `awk`
  - **Write** not `echo >`, `cat <<EOF`
  - Narrow exception: a single mechanical, identical edit applied across many files (e.g. a project-wide import-path or symbol rename) may use one `sed`/`perl -pi` pass when per-file `Edit` would mean dozens of calls. Only when the replacement string is identical everywhere and the result is verified afterward by a compiler/linter/test (e.g. `go build ./...`) or a `Grep` confirming zero stragglers ‚Äî never for context-dependent edits.
- Never chain Bash commands with `&&`, `||`, or `;` ‚Äî run each command as a separate Bash tool call. This includes `cd`, `git`, multi-step inspection (`git status` + `git diff` + `git log`), and any other composite. Reasons: the user can only approve/deny the chain as a whole, intermediate failures are hard to attribute, and an unintended state after a partial run (e.g., a worktree created from stale master) is harder to recover from. Parallel independent calls in one message are fine; chaining in one shell command is not.
- Never use command substitution (`$(...)` or backticks) in Bash tool calls ‚Äî these require explicit permission every time. Instead, capture the output in a prior step and use it directly, or use built-in tools (Glob, Grep, Read) to achieve the same result.
- The Bash tool runs under interactive zsh, so `!` triggers history expansion and a backslash gets inserted into the literal text ‚Äî *even inside single quotes, quoted heredocs (`<<'EOF'`), and arrays*. Writing `feat!: x` to a file via `cat`/`echo` silently produces `feat\!: x` (confirm with `od -c`). This corrupts test fixtures, sample commit titles, and regexes containing `!`. To create files whose content contains `!`, use the **Write** tool, not shell redirection. When testing a regex against such data, read the data from a Write-created file rather than inlining it. This also bites CLI *arguments*, not just file content: `gh pr create/edit --title "feat!: ‚Ä¶"` lands a literal `feat\!:` in the PR title (breaks Conventional-Commit/semver parsing). `git commit -m` happens to survive it, but the `gh` title arg does not. **Do not work around it by creating the PR with a placeholder title and `PATCH`-ing the real title in afterward** ‚Äî a ruleset "Validate PR title" check validates the title on the `opened` event and won't re-run on the rename, so the placeholder is what gets validated and the failure sticks (see `git.md`; learned on `terraform-modules-volcano#160`). Instead set the real title **at creation** via the API, reading it from a Write-created file: `gh api repos/<o>/<r>/pulls -f head=<branch> -f base=<base> -F draft=true -F title=@<titlefile> -F body=@<bodyfile>`. Verify with `gh pr view <n> --json title --jq .title | od -c`.
- Use LSP diagnostics and go-to-definition to understand and troubleshoot code ‚Äî don't guess at types, imports, or call sites by grepping
- When investigating errors: check LSP/compiler output first, then read the relevant source ‚Äî don't shotgun-grep the codebase
- Before asserting that a tool or system **can't** or **won't** do something ‚Äî especially before writing it into long-term memory (the work wiki) or telling the user as fact ‚Äî verify against a concrete artifact: a real PR, CI run, log line, or command output that shows the actual behaviour. Extrapolating a confident negative from a *related* note or general knowledge is how wrong "facts" get recorded. (Learned writing "Renovate won't regenerate `.terraform.lock.hcl`" into the wiki, extrapolated from a manual-regen memory; an existing Renovate PR ‚Äî `terraform-devplat-volcano#434` ‚Äî showed it updates the lock with the full multi-platform hash set. The PRs were available to check the whole time.)
- When a request says "all uses" / "everywhere" / "across the `<X>` dirs" (a plural or a glob), enumerate the full match set first (`Grep` the symbol/module path across the dir glob) and reflect the scope back ‚Äî "that's N refs across M dirs, all of them?" ‚Äî before editing. Don't collapse a plural/glob ("the devplat-pub dirs") to the single instance you happen to be looking at. (Learned scoping "all uses in staging dirs" to one file when the user meant every `devplat-pub-*` state dir.)
- Grafana: always use the Grafana MCP server tools for querying dashboards, logs, metrics, alerts, and incidents ‚Äî never curl or WebFetch the Grafana web UI
- Grafana: when authoring PromQL/LogQL that will land in a dashboard or alert, validate it via `query_prometheus`/`query_loki_logs` first ‚Äî substitute concrete sample values for `${var}` interpolation. Catches regex escaping (`\.` in double-quoted strings), `label_replace` duplicate-labelset errors, and rate-vs-counter mismatches before the dashboard ships.
- When investigating alerts, crash loops, or performance issues: invoke `kognic-devops:investigate` first ‚Äî it has LogQL/PromQL patterns and datasource UIDs that prevent wasted queries
